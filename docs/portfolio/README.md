# Material de portfólio

Esta pasta receberá somente material produzido com dados fictícios.

## Capturas previstas

1. `dashboard-admin.webp`: visão geral com valores e nomes fictícios;
2. `gestao-alunos.webp`: listagem sem telefone, e-mail ou dados reais;
3. `portal-aluno.webp`: perfil de uma persona criada para a demonstração;
4. `arquitetura.webp`: versão visual do fluxo documentado no README raiz.

## Vídeo

O vídeo deve ter entre 60 e 90 segundos e mostrar:

1. problema atendido;
2. login administrativo;
3. dashboard e gestão financeira manual;
4. portal do aluno;
5. arquitetura backend e principais controles de segurança.

## Checklist de anonimização

- usar nomes, e-mails, telefones e valores fictícios;
- não mostrar domínio, IP, logs, tokens, cookies ou variáveis reais;
- não gravar telas da cliente ou banco de produção;
- revisar cada frame antes da publicação;
- manter a demonstração identificada como ambiente de portfólio.
